# CLAUDE.md — Prebid Documentation Site

## Project Overview

This is the Prebid documentation site, actively migrating from **Jekyll** to **Docusaurus 3.x**. Both the old Jekyll structure and new Docusaurus structure coexist in the repo during migration.

- **Production URL**: https://docs.prebid.org
- **Branch**: `docusaurus` (migration work), `master` (main/production)
- **Docusaurus version**: 3.9.2
- **Node requirement**: >=20.0

## Commands

```bash
npm run start    # Dev server with hot reload
npm run build    # Production build (validates broken links)
npm run clear    # Clear Docusaurus cache
npm run serve    # Serve production build locally
```

## Docusaurus Routing

The site uses 5 documentation plugins with separate routing:

| Plugin | Content Path | Route Base | Sidebar |
|---|---|---|---|
| Main docs | `docs/content` | `/` | `sidebars.ts` |
| Prebid.js | `docs/dev-docs/prebidjs` | `/dev-docs/prebidjs` | `sidebars.pbjs.ts` |
| Prebid Server | `docs/dev-docs/prebid-server` | `/dev-docs/prebid-server` | `sidebars.pbs.ts` |
| Prebid Mobile | `docs/dev-docs/prebid-mobile` | `/dev-docs/prebid-mobile` | `sidebars.pbm.ts` |
| Tools | `docs/dev-docs/tools` | `/dev-docs/tools` | `sidebars.tools.ts` |

All sidebars use `{ type: 'autogenerated', dirName: '.' }` — directory structure determines navigation.

## Migration Rules

When migrating files from Jekyll to Docusaurus:

1. **Always use `git mv`** — preserves git history
2. **Frontmatter**: Remove `layout:` and `sidebarType:` lines. Keep title, description, and metadata.
3. **TOC markers**: Remove `{: .no_toc}`, `* TOC`, `{:toc}` (Docusaurus auto-generates)
4. **CSS classes**: Remove `{: .table .table-bordered .table-striped }` and similar
5. **Links**: `{{ site.baseurl }}/path` → `/path`, remove `.html` extensions from internal links
6. **Images**: `{{ site.baseurl }}/assets/images/...` → `/images/...` (static/ dir already has copies)
7. **Alert includes with string content**: `{% include alerts/alert_warning.html content="..." %}` → `:::warning\n...\n:::`
8. **Alert includes with captured variables**: See "Tricky Patterns" below
9. **Markdown file includes**: `{% include dev-docs/loads-external-javascript.md %}` → inline as admonition or React component
10. **Complex includes**: Use `<IncludeTodo />` placeholder (requires `.mdx` extension)
11. **Liquid logic** (`{% assign %}`, `{% for %}`): Cannot be auto-migrated, needs manual conversion
12. **File extension**: Only rename `.md` → `.mdx` when React components are imported
13. **Build validation**: `npm run build` must pass after each batch

## Tricky Migration Patterns

### Capture + Alert pattern (~36 instances in 21 files)

Jekyll files use `{% capture %}` to build content, then pass the variable to an alert include:
```liquid
{% capture noteContent %}
Some **markdown** content with [links](/path).
{% endcapture %}
{% include alerts/alert_note.html content=noteContent %}
```
**Conversion**: Extract the captured content and wrap it in a Docusaurus admonition:
```markdown
:::note
Some **markdown** content with [links](/path).
:::
```

### Files requiring manual conversion (cannot be scripted)

These files use `site.pages` queries and Liquid loops to generate dynamic content:
- `dev-docs/pbs-bidders.md` — queries bidder pages, uses `{% for %}`, `{% unless %}`, `{% case %}`
- `dev-docs/modules/index.md` — queries module pages with `site.pages`
- `dev-docs/modules/userId.md` — queries userid pages with `site.pages`
- `dev-docs/modules/consentManagementUsp.md` — queries bidders for USP support
- `dev-docs/modules/consentManagementGpp.md` — queries bidders for GPP support

These need custom MDX components or static data to replace the Liquid logic.

### Markdown file includes (17 files)

Files that include reusable markdown fragments from `_includes/dev-docs/`:
- `loads-external-javascript.md` (9 files) → inline as `:::warning Disclosure` admonition
- `vendor-exception.md` (4 files, sometimes with `gvlId` param) → inline as `:::warning` admonition
- `native-assets.md` (2 files) → `<IncludeTodo />` or new component
- `native-image-asset-sizes.md` (2 files) → `<IncludeTodo />` or new component

### One HTML file in dev-docs

`dev-docs/requirements/tcf2/PrebidSupportforEnforcingTCF2.html` — needs manual review.

## CI/CD

- **GitHub Actions**: `.github/workflows/ci.yml` runs **markdownlint** on changed `.md` files in PRs
- **Markdown linting config**: `.markdownlint.json` (some rules disabled), `.markdownlintignore` (excludes `_includes`)
- **Code path notifications**: `.github/workflows/code-path-changes.yml` sends alerts on certain file changes

## Key Config Details

- `onBrokenLinks: "throw"` — build fails on any broken link (strict)
- `onBrokenMarkdownLinks: "ignore"` — temporarily relaxed during migration
- Blog is disabled (`blog: false`)
- Custom `tocPlugin` in `_plugins/toc-plugin.ts` generates `bidders.json` from docs with `layout: bidder` frontmatter, output to `docs/dev-docs/prebidjs/bidders.json`
- Navbar has version dropdowns for Prebid.js, Prebid Server, Prebid Mobile, and Tools
